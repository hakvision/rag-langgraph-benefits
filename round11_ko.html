round10 structure

Round10 원본 구조를 제대로 적어두면

질문 분류 → question_type_router → adaptive query plan → retrieval + rerank → retrieval judge → rescue(if needed) → support stitch → evidence distill → citation answer → answer refine(if needed) → 답변

조건부 규칙

  • simple_fact면 짧은 focus branch
  • abstract_why면 rewrite + step-back
  • multi_part면 subquery decomposition
  • retrieval quality 낮으면 14B rescue 1회
  • support coverage 비면 관련 섹션 청크 stitch
  • answer score 낮으면 refine 1회

실험 해석법

  • 각 ablation은 round10에서 해당 노드만 제거한 버전이다.
  • 점수 하락폭이 클수록 그 노드가 효과적이었다는 뜻이다.
  • flow는 scanability 우선으로 간단한 화살표 형태로 보여준다.
