round1~round9를 짧은 화살표 flow로 다시 정리

이번 버전은 복잡한 도식 대신, 각 라운드를 “무엇을 넣었는지” 중심의 한 줄 플로우로 보여준다. 즉 round1은 뭐였다 → round2에서 뭐가 추가됐다 → round8에서 왜 확 좋아졌는지를 바로 읽게 만든 요약본이다.

source doc: AWS Korean RAG mock pipeline best: round8 / round9 = 86.3
메인 허브 텍스트 요약 최신 round9
how to read
  • 굵은 화살표 줄만 봐도 각 라운드의 핵심 구조를 이해할 수 있게 만들었다.
  • 점수는 그 라운드 평균 judge score이고, 색으로 직전 라운드 대비 상승/하락을 표시했다.
  • 핵심만 남겼기 때문에 classify 같은 공통 시작 단계는 필요할 때만 넣고, “새로 추가된 것” 위주로 적었다.
round 1

Baseline 시작점

score 52.9 기준선
simple flow
질문 분류 → 검색 → 품질체크 → LLM → 답변
가장 기본형. 일단 검색하고, 결과가 너무 약하면 품질체크 후 라우팅/재시작하는 구조였다.
round 2

Reranker 추가

score 53.8 +0.9
simple flow
질문 분류 → 검색 → 리랭커 → 품질체크 → LLM → 답변
검색 결과 순서를 한 번 더 정리하는 reranker를 넣어서, 바로 답변에 먹이는 문서 품질을 조금 올렸다.
round 3

Hybrid retrieval

score 64.3 +10.5
simple flow
질문 분류 → lexical 검색 + semantic 검색 → RRF 합치기 → 리랭커 → LLM → 답변
여기서 큰 개선. 검색을 하나만 쓰지 않고 둘로 가져와 합친 다음 rerank해서 retrieval coverage를 키웠다.
round 4

Judge 분리

score 67.5 +3.2
simple flow
hybrid 검색 → 리랭커 → retrieval judge → LLM → answer judge → 답변
검색이 맞는지 보는 judge와, 답변이 괜찮은지 보는 judge를 나눠서 어디서 문제 나는지 더 명확히 보게 만들었다.
round 5

프롬프트/쿼리 개선 시작

score 62.6 -4.9
simple flow
원문 질문 → query rewrite → step-back 질문 → 검색 → LLM → 답변
질문 자체를 더 검색 친화적으로 바꾸는 실험. 아이디어는 좋았지만 이번 세팅에선 점수는 잠깐 내려갔다.
round 6

Subquery 분해

score 63.9 +1.3
simple flow
질문 계획 → subquery A/B/C → 각각 검색 → multi-query fusion → LLM → 답변
질문을 쪼개서 따로 검색한 뒤 다시 합치는 구조. 추상 질문에서 필요한 근거를 조금 더 잘 모으려는 단계였다.
round 7

Evidence distill 추가

score 65.7 +1.8
simple flow
질문 계획 → 검색/합치기 → 리랭커 → 근거 압축 → LLM → 답변
검색 문서를 그대로 던지지 않고, 답변 전에 핵심 근거만 추려서 LLM이 덜 흔들리게 만들었다.
round 8

답변 템플릿 고정

score 86.3 +20.6
simple flow
질문 계획 → 검색/합치기 → 근거 압축 → citation 템플릿 LLM → 답변
여기가 핵심 점프. 답변 형식을 citation-constrained template로 강하게 고정하자 judge score가 크게 뛰었다.
round 9

Self-refine 루프

score 86.3 +0.0
simple flow
질문 계획 → retrieval rescue → 근거 압축 → citation 템플릿 LLM → self-refine → 답변
스스로 한 번 더 고치는 refine 단계를 붙였지만, 평균 점수는 round8과 같아서 효율 면에서는 round8이 더 깔끔했다.
