what this proves

A안에서 지금 확인한 것

  • 최종 RAG가 실제로 어디서 branch 되는지 정리했다.
  • judge_retrieval와 judge_answer가 별개 gate라는 점을 시각적으로 분리했다.
  • retrieval rescue와 answer refine가 항상 실행되는 노드가 아니라 조건부 노드라는 점을 드러냈다.
  • 질문 유형에 따라 adaptive_query_plan 내부 폭이 달라진다는 사실을 별도 branch 힌트로 표현했다.
what this does not prove

아직 아닌 것

  • 실제 LangGraph runtime이나 LangSmith trace는 아직 연결하지 않았다.
  • 노드별 입력/출력 state serialization은 아직 정의하지 않았다.
  • 현재 페이지는 mock provider mode 기준 구조도이며, 실백엔드 추론 latency를 나타내지 않는다.
  • 다음 단계 B안에서 StateGraph 래퍼를 만들어야 질문별 live trace를 볼 수 있다.
